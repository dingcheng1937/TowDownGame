# TowDownGame 项目学习指导文档

## 一、项目概述

本项目是基于 **Godot 4.x** 引擎开发的 2D 俯视角射击游戏，采用 **GDScript** 编写。

**核心系统**:
- 玩家系统：移动、冲刺、战斗
- 武器系统：射击、换弹、配件
- 怪物系统：AI追踪、伤害处理
- 数据系统：全局状态管理

**目录结构**:
```
TowDownGame/
├── game/          # 游戏核心逻辑
│   ├── hero/      # 玩家角色
│   ├── guns/      # 武器系统
│   ├── bullets/   # 子弹系统
│   ├── monster/   # 怪物系统
│   └── reward/    # 奖励系统
├── ui/            # 用户界面
├── autoload/      # 全局单例
└── Sprites/       # 图像资源
```

## 二、核心架构

### 2.1 自动加载单例

| 单例 | 职责 |
|------|------|
| `PlayerData` | 玩家数据（HP、弹药、等级） |
| `Utils` | 工具函数、资源预加载 |
| `MapServer` | 地图管理 |

### 2.2 信号驱动架构

项目使用 Godot 信号系统实现组件解耦：
- `PlayerData.onHpChange` - 血量变化通知
- `PlayerData.onPlayerDeath` - 玩家死亡通知
- `Utils.onGameStart` - 游戏开始通知

## 三、入门学习路线

### 阶段1：熟悉项目（1-2天）
1. 运行游戏，体验核心玩法
2. 阅读 `PlayerData.gd` 了解数据结构
3. 阅读 `Hero.gd` 了解玩家控制

### 阶段2：核心系统（3-5天）
1. 理解武器系统 `BaseGun.gd`
2. 理解怪物系统 `BaseMonster.gd`
3. 实现简单功能扩展

### 阶段3：进阶开发（1周+）
1. 扩展新功能（成就、存档）
2. 优化性能
3. 主题改造

## 四、关键文件速查

| 文件 | 路径 |
|------|------|
| 玩家控制器 | `game/hero/Hero.gd` |
| 武器基类 | `game/guns/BaseGun.gd` |
| 怪物基类 | `game/monster/BaseMonster.gd` |
| 玩家数据 | `autoload/PlayerData.gd` |
| 工具函数 | `autoload/Utils.gd` |

## 五、开发规范

- **类名**: PascalCase（如 `BaseGun`）
- **变量/函数**: snake_case（如 `bullet_speed`）
- **信号**: on_xxx（如 `onHpChange`）
- 使用 `@export` 暴露可配置参数
- 使用信号进行组件通信
